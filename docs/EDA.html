Présentation de la base de données

La base de données exploitée provient de Kaggle, une plateforme de référence pour l’accès à des jeux de données publics. Elle concerne le domaine de l’assurance santé et se caractérise par les éléments suivants :

  • Taille de l’échantillon : 1 337 observations (après suppression du doublon) représentant des individus assurés.
  • Nombre de variables : 7 colonnes comprenant les caractéristiques des assurés ainsi que la variable cible.
  • Qualité des données : Le jeu de données est complet et ne présente aucune valeur manquante sur l’ensemble des observations.

Analyse bivariée

Cette section explore l'influence des différentes caractéristiques sur variable cible (charges) afin d'identifier les principaux leviers de coût :


  • Variable Fumeur : C'est le facteur le plus discriminant. Les fumeurs présentent des charges nettement plus élevées, avec une rupture nette entre les deux groupes.
  • Âge et Charges : Il existe une corrélation positive claire ; le coût de l'assurance augmente de manière constante et linéaire avec le vieillissement.
  • IMC (BMI) et charges : L'impact de l'IMC est particulièrement marqué chez les fumeurs. On observe que les individus fumeurs ayant un IMC > 30 subissent les charges les plus extrêmes.
  • Nombre d'enfants et charges : On note une légère augmentation des charges pour les assurés ayant 2 ou 3 enfants, bien que l'impact soit moins prononcé que le tabagisme.
  • Sexe et charges : Ces variables montrent une influence marginale sur le montant moyen des primes, indiquant une tarification relativement équitable sur ces critères.
  • Région et charges : Ces variables montrent une influence marginale sur le montant moyen des primes, indiquant une tarification relativement équitable sur ces critères.

Analyse de la variable cible

La variable charges représente les coûts de santé individuels facturés par l'assurance. Son analyse est cruciale car elle constitue l'objectif de prédiction de nos modèles :

  • Assymétrie : La distribution est fortement étalée vers la droite (skewed), avec une majorité de primes inférieures à 15 000$.
  • Valeurs Extrêmes : Le boxplot révèle de nombreuses valeurs atypiques au-delà de 35 000$.
  • Normalisation : La transformation en log_charges permet d'obtenir une distribution plus "normale" (en cloche), facilitant l'apprentissage pour certains modèles de régression.

    En raison de son étalement, une transformation logarithmique (log_charges) a été testée pour normaliser la distribution et améliorer la précision des modèles de régression.
